请给 ClawClau 系统添加「进度汇报」功能。这是一个需要长期记住并内置到系统中的能力。

## 背景
当小八通过 claude-spawn.sh 派发一个耗时任务后，用户希望每隔 N 分钟自动收到一条进度汇报，而不是等任务结束后才看到结果。之前没有这个能力，小八只能手动检查，效果很差。

## 需求
1. 修改 claude-spawn.sh，增加可选参数 --interval <秒数>，表示每隔多少秒汇报一次进度
2. 在后台 monitor 循环中（或新建一个 parallel progress-reporter），定期读取日志文件的最新内容，提取进度信息
3. 通过 openclaw system event 发送进度汇报，格式如：'[进度] 任务 xxx: 已运行 3 分钟，日志最新输出：...'（截取日志最后几行，去重）
4. 默认不汇报（向后兼容），只有显式传了 --interval 才启动进度汇报
5. 更新 SKILL.md 文档，记录这个新参数的用法

## 技术约束
- 日志文件路径在 active-tasks.json 里能查到
- openclaw system event --text 'xxx' --mode now 可以发送通知
- 进度汇报应该是去重的：如果上次汇报后日志没有新内容，就跳过
- 汇报时提取日志中有用的信息（比如 [进度 X/6] 这样的标记），不要原样搬 ANSI escape codes

## 交付
1. 修改 claude-spawn.sh
2. 更新 SKILL.md
3. 用一个测试任务验证进度汇报是否正常工作（--interval 30，跑个简单任务，确认中间有进度通知发出来）
