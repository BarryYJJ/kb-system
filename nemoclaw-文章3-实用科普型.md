# 用了半年 OpenClaw，NVIDIA 给它加了一把安全锁

---

你有没有想过，你每天让 AI Agent 帮你读文件、查日历、改代码的时候，它其实拥有的权限，比你公司大部分实习生都大？

这不是玩笑。

2026 年最火的开源项目 OpenClaw，GitHub 上 20 万+ stars，已经成了 AI Agent 的事实标准框架。开发者们用它搭各种自动化工具，从写代码到管数据库，无所不能。

但有个问题一直悬在头上——**安全**。

上周，Jensen Huang 在 GTC 2026 上发布了一个东西，试图解决这个问题。

它叫 **NemoClaw**。

---

## 一、先说问题：你的 Agent，其实在"裸奔"

OpenClaw 有多火，就不用多说了。但火归火，有个事实很多人选择性忽略：

**你的 Agent 能访问你的文件系统、日历、数据库，甚至能发网络请求。**

这意味着什么？

意味着一个恶意的 skill（你可以理解为 Agent 的"插件"），可以悄悄读你的 SSH 密钥，把你的代码传到外面，甚至在你的机器上执行任意命令。这在安全圈有个专业名词叫 RCE——远程代码执行，是安全漏洞里最危险的那一档。

社区已经发现了**约 900 个恶意 skill**，暴露实例超过 **13.5 万个**。

而目前的安全机制是什么呢？

**skill 作者自觉 + 用户手动审查。**

对，就跟早年安卓应用商店一样——上架全靠自觉，安不安全全看用户自己有没有火眼金睛。

这在个人玩玩的时候问题不大。但当企业想把 Agent 用到生产环境，这就是一颗定时炸弹。

---

## 二、NemoClaw：不是新选手，是一套防弹衣

先澄清一个最大的误解：**NemoClaw 不是 OpenClaw 的竞品。**

它是给 OpenClaw 套上的一层安全壳。

你可以这么理解：OpenClaw 是一辆性能很猛的赛车，NemoClaw 是给这辆车加了安全带、气囊和防滚架。车还是那辆车，但撞了不至于要命。

NemoClaw 由三个组件组成，我用大白话解释一下：

**1. OpenShell = 笼子**

把你的 Agent 关在一个隔离的环境里跑。它能做什么、不能做什么，你说了算。（技术上就是 Docker 容器，后面细说。）

**2. Nemotron = 大脑**

NVIDIA 自家的开源大模型，专门为 Agent 场景优化。重点是：**免费**。

**3. NemoClaw = 套装**

把上面两个东西打包好，一键安装，开箱即用。你不需要自己折腾怎么把它们拼在一起。

简单说，NVIDIA 做了一件事：**帮你把安全和模型都准备好了，你只管用。**

---

## 三、OpenShell：这个"笼子"到底怎么关的？

OpenShell 是整套方案里最硬核的部分，也是真正值得关注的技术。

我用一个比喻来解释。

想象你雇了一个特别能干的助手（Agent），让他在你的办公室里工作。以前的做法是：把办公室钥匙给他，然后……就没有然后了。他能开任何柜子，能用你的电话打任何号码，能把任何文件带出去。

OpenShell 的做法是：**给他一间单独的工作间，装上单向玻璃，锁上门，然后精确控制他能接触什么。**

具体来说：

**网络管控：默认全部拦截。**

Agent 所有的出站网络请求，默认返回 403（禁止访问）。你需要通过一个 YAML 配置文件，逐条审批它能访问哪些地址。

而且控制粒度非常细——不只是"能不能访问某个网站"，而是精确到 HTTP 方法和路径。比如你可以设置：允许 GET 请求读取某个 API 的数据，但禁止 POST 请求往里写东西。

就像你允许助手查看客户名单，但不允许他往里添加新客户。

**文件系统：只给指定目录的钥匙。**

Agent 只能读写你明确指定的路径。你的 SSH 密钥、系统配置文件？碰都碰不到。

**进程权限：不能提权。**

Agent 在容器里没法获取更高的系统权限。就算它想"越狱"，也翻不出这道墙。

**模型 API 调用：被劫持到受控后端。**

Agent 调用大模型的请求，会被路由到一个受控的后端，而不是直接连外网。这意味着即使 Agent 被恶意 skill 操控，它也没法偷偷调用别的模型做坏事。

**凭证保护：密码不落盘。**

所有的 API Key、密码之类的敏感信息，以环境变量的方式注入，不会写到磁盘上。Agent 能用，但偷不走。

**策略热加载：改规则不用重启。**

安全策略改了？不需要停机。实时生效。

这套东西的设计哲学很清晰：**默认不信任，按需开放。**

## 四、实际体验：能用，但有点糙

说完原理，说说实操。

目前 NemoClaw 的安装流程是这样的：

**前置要求：**
- Linux Ubuntu 22.04 以上（**Mac 和 Windows 暂时不支持**，Mac 用户先别激动）
- Node.js 20+
- Docker
- OpenShell

**安装：**

```bash
git clone <仓库地址>
./install.sh
```

安装完成后，终端会显示沙箱名称、模型名称和运行命令。

**连接沙箱：**

```bash
nemoclaw my-assistant connect
```

连进去之后，你可以用 `openclaw tui` 打开聊天界面，也可以用 `openclaw agent` 通过命令行发消息。

体验下来的感受：**能跑，但确实是 Alpha 阶段。**

官方自己也说了：expect rough edges（预期会有毛边）。

如果你是那种喜欢尝鲜、不怕折腾的开发者，现在就可以玩。如果你是那种"我只想安安静静用工具"的人，建议再等等。

## 五、Nemotron 3 Super：开源模型的一匹黑马

NemoClaw 里内置的模型叫 **Nemotron 3 Super**，这个模型本身就值得单独说说。

几个关键数字：

- **120B 总参数，只激活 12.7B**

这是 MoE（混合专家）架构。你可以理解为一家 120 人的公司，但每次处理任务只安排 12 个最合适的人上场。不养闲人，效率极高。

- **PinchBench 85.6%**

这是专门测 Agent 能力的评测。开源模型里第一名，甚至超过了 Claude Opus 4.5。

- **SWE-Bench Verified 60.47%**

这是测软件工程能力的标准评测。开源第一。

- **API 速度 442 tokens/秒**

非常快。

- **API 完全免费**

通过 OpenRouter 或 build.nvidia.com 调用，不花钱。

- **1M 上下文窗口**

能处理超长文本。

- **本地部署需要 64GB 内存**

Mac 用户可以用 MLX 量化版，速度约 33.7 tok/s，能用但不快。

但也要说实话：**Nemotron 3 Super 不是万能的。**

在纯知识推理和复杂逻辑任务上，它还打不过 Claude Opus 4.6。但在 Agent 场景——就是让模型去操作工具、执行任务、跟环境交互——它特别稳。

**模型的好坏，要看用在哪。**

## 六、三个模型，怎么选？

既然提到了模型，顺便给一个决策参考：

| 模型 | 价格 | 强项 | 适合场景 |
|------|------|------|----------|
| **Nemotron 3 Super** | 免费 | Agent 操作最稳 | 日常 Agent 任务，默认选择 |
| **GLM-5 Turbo** | $1.2/$4 per MTok | 中文极强，性价比高 | 中文场景、预算敏感 |
| **Claude Opus 4.6** | $5/$25 per MTok | 最强推理能力 | 复杂逻辑、关键决策 |

我的建议是：**别只押一个，三者配合用。**

日常任务扔给 Nemotron（反正免费），中文内容用 GLM-5 Turbo，遇到真正需要深度思考的问题再上 Claude Opus。

## 七、NVIDIA 到底想干什么？

到这里，有必要想一个更大的问题：**NVIDIA 为什么要做这件事？**

答案藏在 NVIDIA 过去十年的套路里。

还记得 CUDA 吗？NVIDIA 把 GPU 编程工具免费送给全世界的开发者。结果呢？所有深度学习框架都建立在 CUDA 之上，然后所有人都得买 NVIDIA 的 GPU。

**免费工具 → 建立生态 → 卖硬件和云服务。**

NemoClaw 是同样的剧本。

模型免费给你，安全工具免费给你。但你用着用着就会发现，你的 Agent 跑在 NVIDIA 的基础设施上了。需要更快的推理？买 GPU。需要企业级部署？上 NVIDIA 的云。

而 **OpenShell 才是这盘棋里真正的核心资产。**

如果 OpenShell 成了 Agent 部署的标准安全方案——就像 Docker 成了应用部署的标准容器方案一样——那 NVIDIA 就占住了 Agent 基础设施的关键位置。

对 OpenClaw 社区来说，这是好事也是隐忧。好事是安全问题终于有人管了。隐忧是，OpenClaw 有可能被"管道化"——变成 NVIDIA 生态里的一个组件，而不是独立的中心。

## 八、适不适合你？

最后，实际的建议：

**如果你是个人开发者：**

现在还是 Alpha 阶段，别急着 all-in。但 Nemotron 的免费 API 值得现在就试试——不装 NemoClaw 也能用模型。等 NemoClaw 稳定了再上车不迟。

**如果你是企业 IT / 架构师：**

这是刚需。没有安全方案，Agent 就没法合规部署。建议现在开始关注和测试 OpenShell，在它成熟之前先摸清它的能力边界。

**如果你是 Mac 用户：**

目前用不了 NemoClaw（只支持 Linux）。可以先用 Nemotron 的 API，等官方适配。

## 写在最后

AI Agent 的发展正在经历一个很有意思的转折点。

过去一年，所有人都在问：Agent 能做什么？能有多聪明？能有多快？

但当 Agent 真正开始进入生产环境，一个更本质的问题浮出水面：**我们怎么确保它不会搞砸？**

NemoClaw 不是最终答案，但它代表了一个正确的方向——**先把门锁好，再让 Agent 干活。**

而 NVIDIA 选择在这个节点切入，用"免费安全工具 + 免费模型"的组合拳做基础设施，这步棋下得很深。

有一句话我很喜欢：**在淘金热中，卖铲子的人往往赚得最多。**

NVIDIA 不挖金子。它卖铲子，还免费送安全帽。
