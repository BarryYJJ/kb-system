## 2. Cognition & Personal Management
Establishes a framework for optimizing cognitive resources and resisting algorithmic exploitation to mitigate decision fatigue.

*   **Architectural Principles**:
    *   **Algorithm Resistance**: Rejection of automated systems as arbiters of information importance. (Ref: `personal/cognition/_index.md`)
    *   **The 90% Rule**: Identifying critical information through natural repetition rather than immediate consumption of "breaking" content. (Ref: `personal/cognition/_index.md`)
    *   **Capacity Constraints**: Conscious filtering of real-time global events to align with biological cognitive limits. (Ref: `personal/context.md`)
*   **Structural Framework**:
    *   **Workflow**: Algorithm Push → Relevance Evaluation → Conscious Filtering → Focus Preservation.
    *   **Scope**: Includes information consumption patterns and mental health management; excludes technical software architecture. (Ref: `personal/context.md`)
*   **Reference Entries**:
    *   **Strategy**: Management of cognitive load and focus. (Ref: `personal/cognition/context.md`)
    *   **Consumption Rules**: Detailed facts on relevance and cognitive limitations. (Ref: `personal/cognition/information_consumption.md`)

## 3. Domain Relationships
The knowledge base maintains a clear separation between technical execution (**Infrastructure**) and the human-centric strategies required to operate within that environment (**Cognition**). While Infrastructure provides the tools for context engineering, the Cognition domain provides the rules for what context is worth preserving.
