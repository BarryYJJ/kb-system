## 知识星球搜索技能 zsxq-search 最终修复（15:28-15:45）

### 关键bug: "全部星球"搜索范围切换无效
- 知识星球搜索面板的"全部星球"按钮是Angular组件（`.search-btn > .all`）
- Playwright的JS `el.click()` 和 `dispatchEvent(new MouseEvent('click'))` 都无法触发Angular事件绑定
- 按钮视觉状态不变（bg仍为白色），placeholder不变，URL仍带groupId
- 结论：从星球页面内部搜索默认锁定"当前星球"，无法通过JS切换

### 修复方案: 直接导航全局搜索URL
- 全部星球：`https://wx.zsxq.com/search/<关键词>`（不带groupId）
- 当前星球：`https://wx.zsxq.com/search/<关键词>?groupId=<星球ID>`
- 已验证有效，搜"GTC"返回了短评&信息和浑水调研Plus两个星球的内容

## ClawClau v1.1.0 — Auto-Monitor on Spawn

### 问题
spawn 乌萨奇任务后没有自动通知机制，之前靠手动 sleep + check 或 cron（被否决）。

### 解决方案：后台 monitor 子进程
在 `claude-spawn.sh` 末尾加入 `(...)& disown` 后台进程：
1. 每10秒 `tmux has-session` 检查任务是否完成
2. session 结束后自动更新 active-tasks.json（status → done/failed + completedAt）
3. 调用 `openclaw system event --text "..." --mode now` 通知小八
4. 任务完成后 monitor 进程自动退出，零残留

### 验证
- monitor-test 任务成功：registry 自动更新 status="done"，日志19字节，进程无残留

### 发布
- ClawHub: `clawclau@1.1.0` (publish ID: k97eg9dfkyg73sfrcjetaxaba5832tes)
- GitHub: commit `b901334` pushed to BarryYJJ/clawclau

### 日志重定向 bug 未解决
乌萨奇跑 monitor-design 任务时日志文件 0 字节（claude 的输出没有被重定向到 log 文件），但 tmux capture-pane 能看到 prompt。wrapper 脚本 `> "$3" 2>&1` 理论上应该能捕获。待排查。
