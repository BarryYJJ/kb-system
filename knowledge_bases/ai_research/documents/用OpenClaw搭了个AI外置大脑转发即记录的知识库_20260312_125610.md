这是一套完全本地化的AI知识库交互系统。能够读报告, 读长文, 读图片, 看公众号文章, 刷B站油管视频, 听播客, 刷社媒(小红书/抖音/微博/..), 读一切链接。

基于OpenClaw + Claude Code 双AI协作。实现往飞书群里扔任何内容或任何链接, 都能自动转成文字→存入本地向量知识库→返回结构化摘要→并用人话总结的全自动工作流。

不需要手动整理, 不需要打标签,不需要分类,扔进去就行,AI 帮你搞定一切。并且实现通过语义搜索的方式与向量知识库交互。

100%本地存储, 零云依赖, 转发你的文件或链接, 得到总结。

1. 直接丢PDF研报,自动提取 + 入库 + 总结

6.大段文字或长截图更简单

半小时内做好正式工作前的信息准备。

并且不让你遗漏任何的看过的信息：

日积月累，形成投研知识库和个人知识库深度内容的沉淀，并且随时可以基于语义检索交互。

而整个过程只有一个动作：转发。
所有你看过的内容，看到一半想要继续看的内容，没时间看的内容。不靠记着一会再看，不靠收藏夹，靠的是一个能理解语义的本地知识库。
这个系统帮你解决所有问题，让你更加专注在思考上，释放你的核心工作时间，提高你的有效创作力。

二、双AI驱动的实现逻辑

这套系统的核心创新是两个AI 的分工协作。

为什么不用一个 AI 搞定？
我最开始就是这么做的——让 OpenClaw 一个人干所有事。结果非常不稳定：规则写了不遵守、该 OCR 的直接描述图片、该入库的反过来问我 要不要存？ 拆成两个之后立刻就稳了：

OpenClaw 只需要做三件事：识别消息类型、把任务扔给 Claude Code、收到结果后写总结。这些它很擅长。
Claude Code 负责所有复杂的技术执行：读 300 行的处理手册、执行 shell 命令、处理错误重试。这些它也很擅长。

分工之后，系统从 时灵时不灵 变成了 稳定输出。本质是让openclaw担任编排层, claude code或者codex担任执行层。在这个过程中, openclaw作为整个项目的产品经理和运营, claude code作为项目的程序员和测试, 而我只是个提要求的用户。

基于此, 我做了两个知识库, 一个针对AI投研场景, 一个针对自我学习迭代。

三、与知识库交互

那要怎么和知识库交互呢？说人话就行了。

在储存时, 处理好的文字经过自动切片、向量化处理，存储在ChromaDB向量数据库中。不需要手动整理, 不需要打标签, 不需要分类, 扔进去就行, AI 帮你搞定一切。

检索时，使用sentence-transformers模型，支持语义检索而非单纯关键词匹配。

这套流程确保了检索的语义准确性——就算你问的问题和原文措辞完全不同, 只要意思相近, 也能找到对应内容。

整套系统能结合知识库进行个性化回答, 可以上传自己的投研框架, 认知模型, 决策方式, 让AI真正成为 了解你 的助手。

四、100% 本地化

这一点对投研人特别重要。

唯一走云端的是 LLM 的推理调用。但你的原始数据——研报全文、投资观点、笔记内容——全部存储在本地，不会留在任何第三方服务器上。

你可以大胆上传你的投研框架、决策方式，认知模型，这些才是你该沉淀成.md格式的东西，openclaw也只是中间态，重要的是在这个过程中沉淀下 1.明确的需求, 2.解决需求的方法，至于中间的回路，总会有新的工具帮你解决，时间问题罢了。
